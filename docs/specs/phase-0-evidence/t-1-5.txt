## Reason for skip

The plan for T-1-5 requires running 4 host-system commands (bootstrap / print / kickstart / bootout)
against the launchd domain `gui/$UID`. The host's PreToolUse guard hook
(`~/.claude/hooks/pretooluse_guard.py`) has an unconditional regex rule
`\\blaunchctl\\b|\\bcrontab\\b|\\bsystemctl\\b` that fires on any Bash command line
containing those literal strings — including pre-task estimate JSON
heredocs, audit-trail rows describing the planned commands, and the actual
test commands themselves.

The rule is unconditional with respect to `CLAUDE_GUARD_MODE` (verified at
S1-D8: switching from `balanced` to `permissive` did NOT lift the block).
Therefore RD-PM dispatch for T-1-5 cannot proceed in this session
without one of:

  (a) editing the hook's regex source to anchor on command-position
      rather than literal-string substring (deferred — out of scope for
      Stage 1; logged as v0.1.8 follow-up via I-019 resolution path d)

      stdout for RD documentation (declined per CEO option choice — too
      much friction for the value)
